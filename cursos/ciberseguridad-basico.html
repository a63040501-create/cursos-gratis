Curso: Ciberseguridad para Desarrolladores

Lección 1: Fundamentos de la Seguridad Web

La seguridad en el desarrollo web se trata de identificar y mitigar riesgos. Un desarrollador seguro es aquel que piensa como un atacante. Aquí cubriremos dos de las vulnerabilidades más comunes:

  • Inyección SQL (SQL Injection): Ocurre cuando un atacante introduce código SQL malicioso a través de una entrada del usuario para manipular tu base de datos.
  • Cross-Site Scripting (XSS): Sucede cuando un atacante inyecta scripts maliciosos en páginas web que otros usuarios ven, robando información o controlando la página.

Aprenderás a usar técnicas como la validación de datos de entrada y el saneamiento de la información para prevenir estos ataques.

Lección 2: Autenticación y Autorización

La autenticación es el proceso de verificar la identidad de un usuario (por ejemplo, con un nombre de usuario y una contraseña). La autorización es el proceso de determinar a qué recursos tiene acceso un usuario una vez que ha sido autenticado.

Veremos cómo almacenar contraseñas de forma segura usando hashing (nunca las guardes en texto plano) y cómo usar JSON Web Tokens (JWT) para gestionar sesiones de usuario de forma segura y sin estado en aplicaciones web.

Lección 3: Cifrado y Protección de Datos

El cifrado es la técnica de convertir datos en un formato ilegible para protegerlos de accesos no autorizados. Es esencial para proteger información sensible como números de tarjetas de crédito o información personal.

Te enseñaremos la diferencia entre hashing y cifrado, y por qué el cifrado es una herramienta indispensable para proteger los datos en tránsito y en reposo. Esto te permitirá garantizar la privacidad y la integridad de la información de tus usuarios.
