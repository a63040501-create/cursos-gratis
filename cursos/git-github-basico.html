Curso: Git y GitHub para Principiantes

Lección 1: ¿Qué es el Control de Versiones?

El **control de versiones** es un sistema que registra los cambios en un archivo o conjunto de archivos a lo largo del tiempo. Piensa en él como una "máquina del tiempo" para tu código. Con el control de versiones, puedes volver a versiones anteriores, ver quién hizo un cambio, y deshacer errores fácilmente. El sistema de control de versiones más popular es **Git**.

**Git** se ejecuta localmente en tu computadora, mientras que **GitHub** es una plataforma en la nube donde puedes subir tus proyectos para compartir, colaborar y hacer un respaldo de tu trabajo.

Lección 2: Conceptos y Comandos Básicos de Git

Para usar Git, necesitas familiarizarte con algunos conceptos clave:

  • **Repositorio (Repo)**: Es la carpeta donde Git rastrea todos los cambios.
  • **Commit**: Es una "instantánea" de tu código en un momento dado. Es como guardar un punto de control.
  • **Branch (Rama)**: Es una versión de tu proyecto que te permite trabajar en nuevas funcionalidades sin afectar la versión principal.

Puedes usar Git desde la terminal de VS Code. Aquí están los comandos más comunes: 


                # Inicializa un nuevo repositorio en tu carpeta
                git init

                # Agrega los cambios para el próximo commit
                git add .

                # Crea un commit con un mensaje descriptivo
                git commit -m "Mi primer commit"

Lección 3: Conectando con GitHub

GitHub es la plataforma perfecta para almacenar tus proyectos. Para conectar tu repositorio local a GitHub, necesitas crear un nuevo repositorio en la página de GitHub y luego usar el siguiente comando para enlazarlo y subir tus cambios. 


                # Enlaza tu repositorio local con uno de GitHub
                git remote add origin https://github.com/tu_usuario/tu_repo.git

                # Sube tus cambios a GitHub (la rama principal)
                git push -u origin main

Aprender a usar Git y GitHub no solo te ayudará a organizar tu trabajo, sino que también es una habilidad fundamental que buscan todas las empresas de tecnología.
