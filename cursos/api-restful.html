Curso: Introducción a las APIs RESTful

Lección 1: ¿Qué es una API?

Una **API (Interfaz de Programación de Aplicaciones)** es un conjunto de reglas que permite que dos aplicaciones de software se comuniquen entre sí. Piensa en una API como un "camarero" en un restaurante. El cliente (el **frontend**) hace un pedido (una **solicitud**) al camarero (la **API**), quien se lo entrega a la cocina (el **backend**). La cocina prepara el plato y se lo da al camarero, que lo entrega al cliente (la **respuesta**).

Las APIs son la base de la mayoría de los servicios web que usas a diario, desde Google Maps hasta las redes sociales.

Lección 2: Conceptos de las APIs RESTful

**REST (Representational State Transfer)** es un estilo de arquitectura para el diseño de APIs. Una API RESTful usa el protocolo HTTP y se basa en recursos (como `/usuarios` o `/productos`). Los datos se envían comúnmente en formato **JSON**, un formato ligero y fácil de leer.

Las APIs RESTful usan los métodos estándar de HTTP para indicar qué acción realizar:

  • `GET`: Para obtener (leer) un recurso.
  • `POST`: Para crear un nuevo recurso.
  • `PUT` / `PATCH`: Para actualizar un recurso.
  • `DELETE`: Para eliminar un recurso.

Lección 3: Conectando el Frontend con el Backend

Para usar una API en el lado del cliente (en JavaScript o React), necesitas una forma de hacer solicitudes HTTP. En JavaScript, la API `fetch` es la herramienta más común para esto.

Una vez que el **backend** expone una URL de API, el **frontend** puede hacer una solicitud para obtener datos. 


                // Ejemplo de una solicitud GET en JavaScript
                fetch('https://api.ejemplo.com/productos')
                    .then(response => response.json())
                    .then(data => console.log(data))
                    .catch(error => console.error('Error:', error));

Este proceso es la clave para crear aplicaciones web de pila completa (full-stack).
